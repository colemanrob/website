Using Open Data to map Bill 148 effects

Posted on March 18, 2018

I covered some interesting changes to minimum wage in my last post. I’m trying to build my skills in the mapping department, so I’m going to create an interactive map of the communities most affected by the change in minimum wage.

Loading the data

The wage data is collected by the Canadian federal government and published as an open dataset available for download. The data is geographically organized by economic region, so we’ll need the shapefile for those boundaries which is also published as an open dataset!

Clean the Wage data

The data has a mix of wage types (hourly, annually) from a variety of sources. For my purposes I’ll take the median wage for those occupations where it’s reported hourly and ignore the annual data.

# clean data
ON_wage_data <- wage_data %>%
  filter(PROV == "ON") %>% #filer ON results only
  filter(`Annual Wage Flag_Salaire annuel` == 0) %>% #remove annual salaries
  filter(!is.na(`Median wage_Salaire Median`)) %>% #remove occupations with missing data
  filter(!`ER Code_Code RE` == "ER35") %>% #remove Ontario total
  mutate(ERUID = str_sub(`ER Code_Code RE`, start=3L)) # remove ER from ER Code so it can be joined


# how many low wage by community

ON_wage_prop <- ON_wage_data %>%
  mutate(low_wage = case_when(
    `Median wage_Salaire Median` <= 14 ~ 1,
    TRUE ~ 0
  )) %>% # add a low wage indicator
  group_by(`ERNAME` = `ER Name_Nom RE`, ERUID) %>%
  summarise(total_occupations = n_distinct(NOC_CNP),
            total_low_wage = sum(low_wage)) %>%
  mutate(prop_low_wage = round(total_low_wage / total_occupations * 100)) # compute the proportion of occupations that are low wage

# results

Build the map

When I first started building the map with leaflet I kept getting this error:

leaflet(merged_ON)

Warning messages: 1: sf layer is not long-lat data 2: sf layer has inconsistent datum (+proj=lcc +lat_1=49 +lat_2=77 +lat_0=63.390675 +lon_0=-91.86666666666666 +x_0=6200000 +y_0=3000000 +datum=NAD83 +units=m +no_defs). Need '+proj=longlat +datum=WGS84'

I used st_transform to change from datum=NAD83 to datum=WGS84

new_ON <- st_transform(merged_ON, "+proj=longlat", "+datum=WGS84")

That managed to allow me to use leaflet without throwing up an errors.

# set colours for cloropleth
bins <- c(0, 8,10, 11, 12, 13)
pal <- colorBin("YlOrRd", domain = new_ON$prop_low_wage, bins = bins)

# create labels
labels <- sprintf(
  "<strong>%s</strong><br/>%d total occupations <br/> %i total low wage occupations",
  new_ON$ERNAME.y, new_ON$total_occupations, new_ON$total_low_wage
) %>% lapply(htmltools::HTML)

map <- leaflet(new_ON) %>%
  addTiles() %>%
  addPolygons(fillColor = ~pal(prop_low_wage),
              color = "white",
              weight = 2,
              opacity = 1,
              fillOpacity = 0.55,
              highlight = highlightOptions(
                weight = 3,
                color = "#666",
                dashArray = "",
                fillOpacity = 0.7,
                bringToFront = TRUE),
              label = labels,
              labelOptions = labelOptions(
                style = list("font-weight" = "normal", padding = "3px 8px"),
                textsize = "15px",
                direction = "auto")) %>%
  addLegend("topright", pal = pal, values = ~new_ON$prop_low_wage,
            title = "% of low wage occupations",
            labFormat = labelFormat(prefix = "%"),
            opacity = 1)
